# ACTA: Autonomous Cognitive Task Assistant

## Обзор проекта

ACTA — это персональная когнитивная платформа нового поколения, предназначенная для управления знаниями, задачами, цифровыми сервисами, устройствами и информационными потоками пользователя посредством естественного взаимодействия.

ACTA не является чат-ботом, отдельной LLM или голосовым помощником. Это интеллектуальная операционная среда, объединяющая различные модели искусственного интеллекта, долговременную память, агентную систему, автоматизацию действий и единый слой управления цифровой экосистемой пользователя.

### Основная цель

Создать персонального цифрового интеллекта, способного понимать контекст, принимать решения, выполнять сложные многоэтапные задачи и взаимодействовать с внешним миром от имени пользователя.

---

## Ключевые особенности

- **Контекст важнее запроса:** ACTA формирует динамическую модель пользователя, учитывая цели, проекты, знания, привычки, предпочтения, историю действий и окружение.
- **Агентная архитектура:** Задачи рассматриваются как процессы, которые ACTA самостоятельно декомпозирует, планирует, назначает специализированных агентов и контролирует результат.
- **Мультимодельность:** Поддержка локальных и облачных AI-моделей разных типов (LLM, VLM, Speech, Embedding).
- **AI Routing:** Автоматический выбор оптимальной модели под задачу с возможностью ручного задания правил.
- **Память:** Несколько типов памяти (эпизодическая, семантическая, персональная, процедурная).
- **Knowledge Graph:** Представление всех объектов системы в виде графа связей для интеллектуального поиска и принятия решений.
- **Интеграция:** Поддержка подключения внешних сервисов и устройств через REST API, GraphQL, Webhooks, MCP и др.
- **Безопасность:** Локальное хранение данных, шифрование, разграничение прав, аудит и контроль доступа.

---

## Технологический стек MVP

- Backend: Python + FastAPI
- Agent Orchestrator: LangGraph
- Memory: PostgreSQL + pgvector
- Knowledge Graph: Neo4j
- Queue: Redis Streams или NATS
- Локальные модели: Ollama / vLLM
- Облачные модели: OpenAI, Anthropic, Gemini через Unified Provider Layer
- Desktop Client: Tauri
- Voice Layer: Whisper + Piper TTS
- MCP Layer для подключения инструментов

---

## Целевая архитектурная модель

ACTA рассматривается как интеллектуальная операционная система поверх цифровой среды пользователя. Пользователь взаимодействует с единой когнитивной сущностью, которая понимает цели, хранит знания, управляет инструментами, автоматизирует процессы и выступает единым интеллектуальным интерфейсом между человеком и цифровым миром.

## Риски и вызовы

- Сложность интеграции множества AI-моделей и агентов
- Высокие требования к безопасности и приватности
- Эффективное управление контекстом и памятью
- Оркестрация агентов и маршрутизация моделей
- Оптимизация мультимодальной обработки

---

## Критерии готовности

- Реализовано ядро с Intent, Context, Reasoning, Planning, Decision и AI Router
- Создана Memory System с минимум Episodic и Semantic памятью
- Разработан базовый Knowledge Graph
- Создан базовый Agent Framework с минимум двумя типами агентов
- Интеграция с минимум одним внешним сервисом
- Обеспечена базовая безопасность
- Создан простой UI с чат-интерфейсом и панелью управления агентами
- Проведено тестирование основных сценариев
- Документирована архитектура и API
