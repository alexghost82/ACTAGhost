# PERFORMANCE_REPORT.md — ACTA

Ранжировано по ROI (выгода / стоимость исправления).

## PERF-1 — Поиск памяти O(N) с полной расшифровкой на каждый запрос — **HIGH ROI**
- **Где**: `memory/store.py:165-216`. `search()` загружает **все** строки пользователя, парсит JSON-токены,
  пересчитывает DF/IDF по всему корпусу при каждом обращении. Вызывается дважды за конвейер
  (`memory_agent.py:21`, `context_agent.py:37`).
- **Влияние**: линейный рост латентности с объёмом памяти; при тысячах записей — секунды на запрос.
- **Фикс**: SQLite FTS5 или инвертированный индекс/кеш IDF; либо pgvector (extras уже есть). Ожидаемый
  выигрыш: с O(N) до ~O(log N), кратное ускорение на больших корпусах.

## PERF-2 — Перезапись всего графа знаний на каждый запрос — **HIGH ROI**
- **Где**: `knowledge_graph_agent.py:36` → `graph.py:37-46`: `save()` сериализует весь граф в JSON и
  переписывает файл при каждом запросе.
- **Влияние**: O(узлы+рёбра) дисковая запись на каждый чат; граф растёт неограниченно (узел на каждый
  `request_id` — `knowledge_graph_agent.py:26`).
- **Фикс**: инкрементальная персистентность/батчинг, или вынести в Neo4j (extras уже есть).

## PERF-3 — Поиск по графу — полный обход узлов — **MEDIUM ROI**
- **Где**: `graph.py:88-100`: `search()` сканирует все узлы, считая совпадения подстрок.
- **Фикс**: индекс по меткам/типам; ограничение размера графа.

## PERF-4 — Единый SQLite-коннект + глобальный RLock сериализует память — **MEDIUM ROI**
- **Где**: `memory/store.py:62-64`, все операции под `with self._lock`. Параллельные воркеры
  (`orchestrator.py:191-194`) всё равно встают в очередь на память.
- **Влияние**: «параллельная» стратегия даёт мало из-за GIL + блокировки БД.
- **Фикс**: пул соединений / WAL-режим / отдельные коннекты на поток.

## PERF-5 — Неограниченный рост буфера аудита в памяти — **MEDIUM ROI**
- **Где**: `security/audit.py:19,31`: `self._buffer` только растёт; `tail()` читает из буфера, не из файла
  (`audit.py:35-37`) → после рестарта история теряется, а в долгом процессе буфер течёт.
- **Фикс**: кольцевой буфер (`collections.deque(maxlen=...)`) + чтение хвоста файла.

## PERF-6 — Конвейер из 15 шагов даже для small-talk — **LOW/MEDIUM ROI**
- **Где**: `orchestrator.py:63-119` всегда исполняет все шаги; для приветствия это лишние LLM-вызовы
  (intent, reasoning, ui — каждый `self.llm(...)`).
- **Фикс**: быстрый путь для `SMALL_TALK`/коротких `QUESTION`.

## PERF-7 — Проверка доступности Ollama по сети на каждом роутинге — **LOW ROI**
- **Где**: `providers/cloud.py:123-130` + `router.py:93-101` дёргает `is_available()` в цикле кандидатов.
- **Фикс**: кешировать доступность с TTL.
