# Архитектура системы ACTA

## 1. Общая архитектура

ACTA построена как интеллектуальная операционная система с модульной микросервисной архитектурой, включающей следующие слои:

- **Core Layer (Интеллектуальное ядро):**
  - Intent Engine — определение намерений пользователя
  - Context Engine — формирование и обновление рабочего контекста
  - Reasoning Engine — логическое построение цепочек рассуждений
  - Planning Engine — генерация планов выполнения задач
  - Decision Engine — выбор стратегий и маршрутизация
  - AI Router — выбор и переключение AI-моделей и агентов

- **Memory System:**
  - Episodic Memory — история событий
  - Semantic Memory — фактические знания
  - Personal Memory — профиль пользователя и предпочтения
  - Procedural Memory — сценарии и автоматизации

- **Knowledge Graph:**
  - Управление графом сущностей и связей
  - Интеллектуальный поиск и анализ

- **Agent Framework:**
  - Набор специализированных агентов (Research, Coding, DevOps, Security, Document, Finance, Communication, Automation)

- **Integration Layer:**
  - Подключение внешних сервисов и устройств через REST API, GraphQL, Webhooks, MCP, локальные процессы

- **Multimodal Layer:**
  - Обработка входных данных (текст, голос, изображения, видео и др.)
  - Генерация выходных данных (ответы, отчёты, код, уведомления)

- **Security Layer:**
  - Локальное хранение и шифрование данных
  - Разграничение прав агентов
  - Журналирование и аудит действий

- **UI Layer:**
  - Веб-интерфейс и десктопный клиент для взаимодействия с системой

## 2. Взаимодействие компонентов

1. Пользователь взаимодействует с UI (чат, голос, панель управления).
2. Core Layer принимает запрос, определяет намерение и формирует контекст.
3. Reasoning и Planning Engines создают план выполнения задачи.
4. Decision Engine выбирает стратегию и маршрутизирует задачи через AI Router.
5. AI Router назначает специализированных агентов для выполнения подзадач.
6. Агенты взаимодействуют с внешними сервисами через Integration Layer.
7. Memory System обновляется на основе результатов и событий.
8. Knowledge Graph используется для интеллектуального поиска и поддержки решений.
9. Security Layer обеспечивает контроль доступа и безопасность.
10. Результаты возвращаются пользователю через UI.

## 3. Модель данных

- **User:** профиль, предпочтения, привычки
- **Project:** цели, задачи, статусы
- **Task:** подзадачи, статусы, назначенные агенты
- **Agent:** тип, состояние, права
- **Service:** внешние интеграции
- **Device:** IoT и локальные устройства
- **KnowledgeItem:** документы, инструкции
- **Event:** история действий и диалогов

## 4. Технологии и инструменты

- Backend: Python + FastAPI
- База данных памяти: PostgreSQL + pgvector
- Knowledge Graph: Neo4j
- Очередь сообщений: Redis Streams или NATS
- Локальные модели: Ollama, vLLM
- Облачные модели: OpenAI, Anthropic, Gemini
- Оркестратор агентов: LangGraph
- UI: Tauri (десктоп), веб-интерфейс
- Голосовой слой: Whisper + Piper TTS
- Протокол межагентного взаимодействия: MCP

## 5. Безопасность

- Локальное хранение пользовательских данных
- Шифрование всех данных в памяти и на диске
- Разграничение прав доступа агентов
- Журналирование всех действий и решений
- Аудит и контроль критических операций
- Возможность работы в полностью оффлайн-режиме

---

Данная архитектура обеспечивает масштабируемость, модульность и безопасность платформы ACTA, позволяя эффективно интегрировать различные AI-модели и сервисы, а также автономно выполнять сложные когнитивные задачи пользователя.
